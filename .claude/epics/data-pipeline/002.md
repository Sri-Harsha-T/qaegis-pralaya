# Task: OpenWeatherMap API client + normaliser

## Description

Create OpenWeatherMap API client within `backend/data_pipeline.py` that fetches real-time weather data for any city and normalises it to match the feature schema used by the VQC model. Provides fallback to synthetic data when API is unavailable.

## Acceptance Criteria
- [ ] Function `get_live_features(city="Bengaluru")` implemented
- [ ] Returns normalised feature dict with keys: temperature_c, wind_kmh, precipitation_mm
- [ ] Falls back to synthetic sample if API unavailable (network/quota/key issues)
- [ ] Completes in < 3 seconds (includes timeout handling)
- [ ] Uses environment variable for API key via `python-dotenv`
- [ ] Proper error handling with logging, no bare except clauses

## Technical Details

- Use OpenWeatherMap Current Weather API: `https://api.openweathermap.org/data/2.5/weather`
- Normalise temperature from Kelvin to Celsius
- Convert wind speed from m/s to km/h
- Extract precipitation from rain.1h field (default 0 if missing)
- Use `httpx` for async HTTP requests with 2-second timeout
- Environment variable: `OPENWEATHER_API_KEY`
- Fallback mechanism returns random sample from synthetic data

## Dependencies

None - this is an independent task that can run in parallel.
