# Task: USGS seismic GeoJSON client

## Description

Create USGS Earthquake API client within `backend/data_pipeline.py` that fetches recent seismic activity data and normalises it for integration with the cascade prediction pipeline. Provides additional real-world trigger data for infrastructure failure scenarios.

## Acceptance Criteria
- [ ] Function `get_seismic_features(region="india")` implemented
- [ ] Returns dict with keys: max_magnitude, event_count_24h, depth_km_avg
- [ ] Fetches data from USGS Earthquake API (GeoJSON format)
- [ ] Completes in < 3 seconds with proper timeout handling
- [ ] Falls back to zero values if API unavailable
- [ ] Filters events by geographic region with configurable bounding box

## Technical Details

- Use USGS Earthquake API: `https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_day.geojson`
- Filter events by lat/lng bounding box for Indian subcontinent
- Extract magnitude, depth, and count metrics from last 24 hours
- Use `httpx` for async HTTP requests with 2-second timeout
- Default bounding box for India: lat 8-37, lng 68-97
- Return normalised values: magnitude (0-10), count (0-100+), depth (0-700km)
