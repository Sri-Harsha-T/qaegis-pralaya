# Task: liboqs verify + Dilithium3 keygen

## Description
Verify liboqs-python installation and implement Dilithium3 key generation. This is the foundation task for the PQC layer that must run first to establish if liboqs is available on the machine. If liboqs is not available, establish the HMAC-SHA256 fallback pattern.

## Acceptance Criteria
- [ ] `python scripts/verify_pqc.py` exits with code 0
- [ ] Script detects liboqs availability and reports status
- [ ] If liboqs available: generates Dilithium3 keypair successfully
- [ ] If liboqs unavailable: logs WARNING and demonstrates HMAC-SHA256 fallback
- [ ] Key generation performance measured and logged
- [ ] Public/private key format documented (hex encoding)

## Technical Details
- Algorithm: `oqs.Signature("Dilithium3")` per ADR-0002
- Never use Kyber (KEM) or other algorithms
- Test key generation + signing round-trip
- Measure performance: target <500ms for sign+verify
- HMAC-SHA256 fallback pattern if liboqs unavailable
- Never silent fail - always log fallback with WARNING level

## Dependencies
- liboqs-python package (optional - fallback required)
- Python 3.11+ with logging
