# Task: IncidentReport + SignedReport dataclasses

## Description
Create dataclass schemas for incident reports and signed reports. These form the core data structures for the PQC signing system and define the exact format that will be cryptographically signed.

## Acceptance Criteria
- [ ] `IncidentReport` dataclass with all required fields
- [ ] `SignedReport` dataclass extending IncidentReport with signature fields
- [ ] UUID generation for report_id
- [ ] ISO 8601 timestamp formatting
- [ ] Location validation (lat/lon bounds checking)
- [ ] Severity validation (1-5 range)
- [ ] JSON serialization/deserialization methods
- [ ] Type hints on all fields and methods

## Technical Details
### IncidentReport schema
- report_id: UUID (auto-generated)
- reporter_id: str
- timestamp: str (ISO 8601 format)
- location: dict with lat, lon, district fields
- incident_type: str
- severity: int (1-5 range)
- payload: dict (free-form additional data)

### SignedReport schema  
- All IncidentReport fields (via inheritance or composition)
- signature_hex: str
- algorithm: str (e.g., "Dilithium3" or "HMAC-SHA256")
- signer_id: str

## Dependencies
- Python 3.11+ with dataclasses
- uuid module for report_id generation
- datetime for ISO 8601 timestamps
