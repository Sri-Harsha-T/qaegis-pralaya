# Task: PQCSigner - sign / verify / serialize

## Description
Implement the core PQCSigner class with Dilithium3 signing/verification and HMAC-SHA256 fallback. This is the main cryptographic component that handles all signing operations with performance requirements.

## Acceptance Criteria
- [ ] `PQCSigner.generate_keypair()` returns (public_key_hex, private_key_hex)
- [ ] `PQCSigner.sign_report(report, private_key_hex)` returns SignedReport
- [ ] `PQCSigner.verify_report(signed_report, public_key_hex)` returns True for valid signatures
- [ ] Tampered report verification returns False (explicit test required)
- [ ] Sign + verify round-trip completes in <500ms
- [ ] HMAC-SHA256 fallback when liboqs unavailable (WARNING logged)
- [ ] Private keys never logged or serialized to disk
- [ ] Proper error handling for invalid keys/signatures

## Technical Details
- Primary: `oqs.Signature("Dilithium3")` per ADR-0002
- Fallback: HMAC-SHA256 with WARNING log when liboqs unavailable
- Key format: hex-encoded strings for JSON serialization
- Performance target: <500ms for sign+verify operations
- Signature verification must detect tampering reliably
- Never silent fallback - always log algorithm selection

## Dependencies
- Task 010: liboqs verification and key generation pattern
- Task 011: IncidentReport and SignedReport dataclasses
- liboqs-python (optional with fallback)
- hashlib for HMAC fallback
