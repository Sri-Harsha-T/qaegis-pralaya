# Task: ResponderKeyRegistry + synthetic responders

## Description
Create a key registry system that manages responder public keys and pre-populate it with synthetic responder data for demo purposes. This simulates the EOC acting as a certificate authority for field responders.

## Acceptance Criteria
- [ ] `ResponderKeyRegistry` class implemented
- [ ] Registry pre-populated with 5 synthetic responders
- [ ] Each responder has: responder_id, public_key_hex, name, department, active status
- [ ] Key lookup by responder_id working
- [ ] Registry persistence to JSON file
- [ ] Registry loading from JSON file
- [ ] Active/inactive responder status management
- [ ] Key rotation capability (add new key, deactivate old)

## Technical Details
### ResponderKeyRegistry interface
- `add_responder(responder_id, public_key_hex, name, department)`
- `get_public_key(responder_id) -> str | None`
- `list_active_responders() -> List[dict]`
- `deactivate_responder(responder_id)`
- `save_to_file(filepath)`
- `load_from_file(filepath)`

### Synthetic responder data
- Fire Department: Captain Smith, Lieutenant Jones
- Police Department: Officer Brown, Sergeant Davis  
- Medical Services: Paramedic Wilson
- Each with generated Dilithium3 or HMAC keys depending on availability
