# Task: FastAPI /sign + /verify endpoints

## Description
Create REST API endpoints for signing and verifying incident reports. These endpoints provide the HTTP interface for the PQC signing system and will be used by field responders and the EOC dashboard.

## Acceptance Criteria
- [ ] `POST /api/pqc/sign` endpoint accepts IncidentReport, returns SignedReport
- [ ] `POST /api/pqc/verify` endpoint accepts SignedReport + public_key, returns verification result
- [ ] Proper HTTP status codes (200, 400, 401, 500)
- [ ] Request/response validation with Pydantic models
- [ ] Error handling for invalid signatures, missing keys, malformed requests
- [ ] API documentation auto-generated by FastAPI
- [ ] CORS headers configured for frontend access
- [ ] Endpoint performance <500ms per ADR requirements

## Technical Details
### /sign endpoint
- Input: IncidentReport JSON + signer_id (header or body)
- Lookup signer's private key from registry
- Sign report using PQCSigner
- Return: SignedReport JSON

### /verify endpoint  
- Input: SignedReport JSON + public_key_hex
- Verify signature using PQCSigner
- Return: {"valid": boolean, "algorithm": string, "signer_id": string}

### Error responses
- 400: Invalid request format, missing fields
- 401: Unknown signer_id, invalid key
- 500: Signing/verification errors, system failures
